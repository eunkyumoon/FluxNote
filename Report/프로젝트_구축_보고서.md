# FluxNote 메모 앱 구축 보고서

## 프로젝트 개요

Create React App(CRA)을 사용하여 반응형 메모 앱을 구축했습니다. 이 앱은 사용자가 노트를 작성, 수정, 삭제하고 검색할 수 있는 기능을 제공합니다.

**프로젝트명**: FluxNote  
**구축일**: 2024년  
**기술 스택**: React 18.2.0, Create React App 5.0.1

---

## 구현된 기능

### 1. 새로운 노트 작성 ✅
- **구현 위치**: `src/App.js`의 `handleCreateNote` 함수
- **기능 설명**: 
  - "새 노트" 버튼 클릭 시 새로운 노트 객체 생성
  - 고유 ID는 타임스탬프 기반으로 생성
  - 생성된 노트는 즉시 편집 모드로 전환
- **데이터 구조**:
  ```javascript
  {
    id: Date.now(),
    title: '',
    content: '',
    createdAt: new Date().toISOString(),
    updatedAt: new Date().toISOString()
  }
  ```

### 2. 기존 노트 수정 ✅
- **구현 위치**: `src/components/NoteEditor.js`
- **기능 설명**:
  - 제목과 내용을 실시간으로 수정 가능
  - 변경사항은 자동으로 저장되며 `updatedAt` 필드 업데이트
  - 로컬 스토리지에 자동 동기화
- **특징**:
  - 실시간 저장 (자동 저장)
  - 수정 시간 추적

### 3. 노트 저장 및 삭제 ✅
- **저장 기능**:
  - **구현 위치**: `src/App.js`의 `useEffect` 훅
  - 로컬 스토리지에 자동 저장
  - 노트 배열이 변경될 때마다 자동 동기화
- **삭제 기능**:
  - **구현 위치**: `src/App.js`의 `handleDeleteNote` 함수
  - 삭제 전 확인 다이얼로그 표시
  - 노트 목록과 편집기 모두에서 삭제 가능
  - 삭제 후 선택된 노트가 삭제된 경우 편집기 초기화

### 4. 제목 또는 내용으로 노트 검색 ✅
- **구현 위치**: `src/components/SearchBar.js` 및 `src/App.js`의 필터링 로직
- **기능 설명**:
  - 검색창에 입력한 키워드로 실시간 필터링
  - 제목과 내용 모두에서 검색 (대소문자 구분 없음)
  - 검색 결과는 노트 목록에 즉시 반영
  - 검색어 초기화 버튼 제공
- **검색 알고리즘**:
  ```javascript
  notes.filter(note => {
    const query = searchQuery.toLowerCase();
    return (
      note.title.toLowerCase().includes(query) ||
      note.content.toLowerCase().includes(query)
    );
  })
  ```

## 컴포넌트 상세 설명

### App.js (메인 컴포넌트)
- **역할**: 전체 앱의 상태 관리 및 레이아웃 구성
- **주요 기능**:
  - 노트 배열 상태 관리
  - 선택된 노트 상태 관리
  - 검색 쿼리 상태 관리
  - 로컬 스토리지 동기화
  - CRUD 작업 처리

### NoteList.js (노트 목록 컴포넌트)
- **역할**: 노트 목록 표시 및 선택 기능
- **주요 기능**:
  - 필터링된 노트 목록 표시
  - 노트 미리보기 (제목, 내용 일부, 날짜)
  - 노트 선택 및 삭제
  - 빈 상태 처리

### NoteEditor.js (노트 편집기 컴포넌트)
- **역할**: 선택된 노트의 편집 기능 제공
- **주요 기능**:
  - 제목 편집
  - 내용 편집 (텍스트 영역)
  - 실시간 저장
  - 노트 삭제
  - 생성/수정 시간 표시

### SearchBar.js (검색바 컴포넌트)
- **역할**: 노트 검색 기능 제공
- **주요 기능**:
  - 검색어 입력
  - 검색어 초기화 버튼
  - 실시간 검색

## 반응형 디자인 구현

### 브레이크포인트
- **데스크톱** (> 768px): 사이드바와 편집기 나란히 배치
- **태블릿** (≤ 768px): 사이드바와 편집기를 세로로 배치, 사이드바 높이 제한
- **모바일** (≤ 480px): 더 작은 폰트 크기와 패딩 적용

### 주요 반응형 기능
1. **레이아웃 변경**: 
   - 데스크톱: 가로 배치 (flex-direction: row)
   - 모바일: 세로 배치 (flex-direction: column)

2. **사이드바**:
   - 데스크톱: 고정 너비 300px
   - 모바일: 전체 너비, 최대 높이 40vh

3. **폰트 크기 조정**:
   - 헤더, 제목, 본문 모두 화면 크기에 따라 조정

## 데이터 저장 방식

### 로컬 스토리지 사용
- **키**: `fluxnote-notes`
- **형식**: JSON 문자열
- **동기화**: 
  - 앱 로드 시 자동 불러오기
  - 노트 변경 시 자동 저장
- **장점**: 
  - 서버 없이 작동
  - 브라우저 간 데이터 유지
  - 빠른 접근 속도

## 설치 및 실행 방법

### 1. 의존성 설치
```bash
npm install
```

### 2. 개발 서버 실행
```bash
npm start
```
앱이 [http://localhost:3000](http://localhost:3000)에서 실행됩니다.

### 3. 프로덕션 빌드
```bash
npm run build
```
빌드된 파일은 `build/` 폴더에 생성됩니다.

## 결론

Create React App을 사용하여 요구사항에 맞는 반응형 메모 앱을 성공적으로 구축했습니다. 모든 필수 기능(생성, 수정, 삭제, 검색)이 구현되었으며, 로컬 스토리지를 활용한 데이터 영속성과 반응형 디자인이 적용되었습니다.

앱은 모바일, 태블릿, 데스크톱 환경에서 모두 잘 작동하며, 직관적인 사용자 인터페이스를 제공합니다.
